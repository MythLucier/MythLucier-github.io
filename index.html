#复杂度分析

复杂度也叫渐进复杂度,包括时间复杂度和空间复杂度，用来分析算法执行效率与数据规模之间的增长关系，可以粗略的表示，越高阶复杂度的算法，执行效率越低。

##1.时间复杂度

###1.1时间复杂度计算方式

int cal(int n) { 
    int sum = 0; 
    int i = 1; 
    for (; i <= n; ++i) { 
        sum = sum + i;
    }
    return sum; 
}

由上面代码例子，时间复杂度未O(n),因为又一个for循环，它在for循环里需要运行n遍，所以n很大时可以忽略其他固定时间的步骤，所以是O(n)

由此类推

int cal(int n) {
    int sum = 0; 
    int i = 1; 
    int j = 1;
    for (; i <= n; ++i) {
        j = 1;
        for (; j <= n; ++j){ 
            sum = sum + i * j;
        }
     } 
}

以上代码的时间复杂度未O(n^2)

时间复杂度计算公式T(n) = O(f(n))

其中T(n)为代码执行时间,f(n)是代码执行总次数,O代表代码的执行时间T(n)和f(n)表达式成正比 (f(n)可以将常数忽略[自己补充的])

###1.2乘法法则

在函数嵌套时使用乘法法则和上面for循环嵌套例子一样

int c1(int n){
    int sum = 0;
    for(int i = 0; i <= n; i++){
        sum += 1;
    }
    return sum;
}
int c2(int n){
    int sum = 0;
    for(int i = 0; i <= n; i++){
        sum += c1(n);
    }
    return sum;
}
int example(int n){
    int sum = 0;
    for(int i = 0; i <= n; i++){
        sum += c2(n);
    }
    return sum;
}
int main(){
    cout << example(n);
}

该例子的复杂度为O(n^3)

###1.3复杂度量级

O(logn) 例子折半查找

O(nlogn) 例子快速排序

###1.4最坏、最好情况时间复杂度

// n表示数组array的长度
int find(int[] array, int n, int x) { 
    int i = 0; 
    int pos = -1; 
    for (; i < n; ++i) { 
        if (array[i] == x) { 
            pos = i; 
            break; 
        } 
    } 
    return pos;
}

由以上例子是查找x元素在数组中的下标，如果x在数组中的下标为0，那么时间复杂度就是最好的情况O(1),
如果是最后一个那就是最坏的情况O(n)

##2.空间复杂度

void print(int n) { 
    int i = 0; 
    int[] a = new int[n]; 
    for(i; i<n; i++){
        a[i] = i;
    }
    for (i; i = 0; --i) { 
        print out a[i] }
    }
}

这里的空间复杂度就为O(n),因为数组a的大小为n
